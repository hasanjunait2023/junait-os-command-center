# Junait OS Operating Playbook

## Founder interaction model
Junait interacts with the system through one command center and one main operator: JAMAL.

## Primary command flow
1. Founder checks dashboard
2. Founder identifies critical issue or target
3. Founder issues BLACKFORGE TASK
4. JAMAL routes the task
5. Relevant role executes
6. Grace / Dan gate risky releases
7. Founder receives concise outcome + next move

## Realtime operating cadence
### Always watch
- infrastructure health
- blocked high-priority tasks
- critical business alerts
- release state
- automation failures

### Daily review
- top 3 priorities
- critical blockers
- urgent follow-ups
- key business snapshots
- content / product / infra statuses

### Weekly review
- shipped work
- business KPI snapshot
- blocked systems
- integration health
- top risks
- next-week priorities

## Dashboard card model
Every dashboard card should display:
- module name
- status
- owner
- key metric
- last update
- next action

## Execution rule
If a module is not yet integrated live, the system should still track it as:
- initialized
- connectable
- planned
- disconnected
- healthy
- warning
- critical
